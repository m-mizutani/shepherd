Reads tickets that already exist in this workspace. Use `ticket_search` to find similar past tickets by title/description substring (optionally filtered by status), then `ticket_get` to fetch a single ticket by UUID or human-friendly seq number. `ticket_get_comments` returns the conversation history attached to a ticket, and `ticket_get_history` returns its status-transition audit log. Useful for finding precedents, recovering prior triage decisions, or quoting earlier discussion when planning the current ticket.
